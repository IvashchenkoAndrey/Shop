## Тестовое задание

Что нужно сделать
1. Создать новую конфигурацию "Магазин".
2. Создать справочник товаров с теми реквизитами, которые Вы посчитаете нужными.
3. Создать регистраторы для фиксации поступления и реализации товаров.
4. Предусмотреть печатные формы для созданных регистраторов.
5. Создать отчет на СКД по проданным товарам за указанный период.
6. Создать отчет на СКД по остаткам товаров.

7*. Создать обработку, которая будет загружать курс доллара в конфигурацию с любого доступного в интернете сервиса по заданному расписанию.

8*. Добавить в отчет из пункта 5 суммы в валюте.

## Как выполнялось задание

1. На первом пункте останавливаться не будем.

2. Создан справочник, стандартные реквизиты "Номер" и "Наименование" + "Артикул"

3. Созданы регистраторы, "Реализация товаров" и "Поступление товаров".
- "Поступление товаров" делает движения с видом приход по регистру "Товары на складах". Фиксирует сроки годности.
- "Реализация товаров" делает движения с видом расход по регистру "Товары на складах". Реализован контроль остатков. Списание происходит по принципу FEFO, товар с наименьшим сроком годности списывается первым, и так по возростанию срока.
- "Реализация товаров" делает движения по регистру "Продажи", дополнительно фиксирует дату документа в измерение "ДатаДляКурса", это пригодится в отчете СКД.

4. При внедрении печатных форм документов внедрил подсистему "Печать" с БСП версии 3_1_11_189. Очень нравится наличие предварительного просмотра и насройки параметров печати. Сам пользуюсь на работе SAP без этих возможностей, часто криво печатает таблицы емких документов.
При запуске обработки "Первое внедрение БСП" помимо подсистемы "Печать" необходимо внедрять еще 3 базовые подсистемы и 1 зависимую, иначе ничего не выйдет
